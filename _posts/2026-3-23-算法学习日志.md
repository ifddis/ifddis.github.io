## 🧠 一、全排列（permutations）

### 1. 用法

```python
from itertools import permutations

permutations([1,2,3])
```

### 2. 复杂度

* 时间复杂度：**O(n!)**
* 空间复杂度：

  * 迭代器：O(n)
  * 转列表：O(n! * n)

### 3. 核心认知

* 本质是枚举所有可能排列
* n 稍大就会爆炸（10! 已经很大）

## 🧩 四、最长连续序列（LeetCode）

### ❌ 错误思路

* 用数组映射值 → 会爆空间

### ✅ 正确思路（核心）

```python
s = set(nums)

for x in s:
    if x - 1 not in s:  # 起点
```

👉 只从“序列起点”开始扩展

---

### 核心思想

* 避免重复计算
* 每个元素最多访问 1 次

👉 时间复杂度：**O(n)**

## 🌧 五、接雨水问题

### 1. DP 解法

```python
left_max[i]
right_max[i]
```

👉 水量：

```text
min(left_max, right_max) - height[i]
```

---

### 2. 双指针（最优🔥）

核心逻辑：

```text
哪边低 → 先算哪边
```

---

### 💡 关键理解（难点）

👉 为什么可以“提前结算”？

因为：

* 左 < 右 → 右边一定能挡住水
* 所以左边水量只由 left_max 决定

---

### 一句话总结

👉 **用局部信息（left_max / right_max）安全计算水量**

## 🪟 六、滑动窗口

### 定义

👉 用左右指针维护一个“动态区间”

---

### 模板

```python
while right < n:
    # 扩展窗口
    right += 1

    while 不满足条件:
        left += 1

    # 更新答案
```

---

### 示例：最长无重复子串

```python
def lengthOfLongestSubstring(s):
    char_set = set()
    left = 0
    res = 0

    for right in range(len(s)):
        while s[right] in char_set:
            char_set.remove(s[left])
            left += 1

        char_set.add(s[right])
        res = max(res, right - left + 1)

    return res
```

---

### 核心思想

👉 **窗口内始终维护“合法状态”**

---

### 复杂度

* 时间：O(n)
* 每个元素最多进出窗口一次
