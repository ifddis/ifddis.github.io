# 🧠 学习日志：滑动窗口 & 字符处理（Python）

## 📌 题目背景

给定两个字符串 `s` 和 `p`，找到 `s` 中所有 `p` 的**异位词子串**的起始索引。

---

## ❌ 初始解法（低效）

```python
for i in range(0, len(s)):
    s1 = sorted(s[i:l])
    temp_p = sorted(p)
    if s1 == temp_p:
        lst.append(i)
    l += 1
```

### 🚨 问题分析

* 每次都：

  * 切片 `s[i:i+k]`
  * 排序 `sorted()`
* 时间复杂度：

```
O(n * k log k) ≈ O(n² log n)
```

👉 **重复计算严重，没有利用已有信息**

---

## 🚀 优化思路：滑动窗口

### 💡 核心思想

> 滑动窗口 = **复用已有结果 + 只更新边界变化**

---

### 🔄 窗口变化过程

```
窗口1: [i ... i+k-1]
窗口2: [i+1 ... i+k]
```

只发生两件事：

* ➖ 移除左边元素 `s[i]`
* ➕ 加入右边元素 `s[i+k]`

👉 中间部分完全复用！

---

## ✅ 优化解法（频率数组）

```python
s_lst = [0] * 26
p_lst = [0] * 26
```

### 🧠 字符映射

```python
ord(c) - ord('a')   # 映射到 0~25
```

---

### 🧩 完整代码

```python
class Solution(object):
    def findAnagrams(self, s, p):
        s_lst = [0 for _ in range(26)]
        p_lst = [0 for _ in range(26)]

        for pi in p:
            p_lst[ord(pi) - ord('a')] += 1

        l = len(p)

        for si in s[0:l]:
            s_lst[ord(si) - ord('a')] += 1

        lst = []

        for i in range(0, len(s) - l + 1):
            if s_lst == p_lst:
                lst.append(i)

            if i + l < len(s):
                s_lst[ord(s[i]) - ord('a')] -= 1
                s_lst[ord(s[i + l]) - ord('a')] += 1

        return lst
```

---

## ⏱️ 复杂度对比

| 方法   | 时间复杂度        | 空间复杂度 |
| ---- | ------------ | ----- |
| 排序法  | O(n·k log k) | O(k)  |
| 滑动窗口 | **O(n)**     | O(1)  |

## 🔍 Python 相关知识点

### 1️⃣ `enumerate()`

```python
for i, v in enumerate(s):
```

* 时间复杂度：O(n)
* 单步操作：O(1)
* 惰性生成，不复制数据

---

### 2️⃣ 字符 ↔ ASCII

```python
ord('a')   # 97
chr(97)    # 'a'
```

| 操作      | 方法      |
| ------- | ------- |
| 字符 → 数字 | `ord()` |
| 数字 → 字符 | `chr()` |

👉 Python 没有 `char` 类型，字符本质是 `str`

---

### 3️⃣ 字符计数（Counter）

```python
from collections import Counter

Counter("aab")  # {'a': 2, 'b': 1}
```

👉 写法简洁，但在高性能场景下不如数组

---

## 🔥 滑动窗口本质总结

> ❌ 每次重算整个窗口
> ✅ 只更新变化部分（边界）

---

## 🧠 进阶优化（减少比较开销）

当前写法：

```python
if s_lst == p_lst
```

👉 O(26) 比较（常数级）

优化思路：

* 使用 `match` / `valid`
* 记录“有多少字符完全匹配”
* 避免整数组比较

👉 可实现更“纯”的 O(n)

---

## 🚀 滑动窗口适用场景

### ✔ 固定窗口

* 异位词查找
* 子串匹配

### ✔ 可变窗口

* 最长无重复子串
* 最小覆盖子串
* 至多 K 个不同字符
