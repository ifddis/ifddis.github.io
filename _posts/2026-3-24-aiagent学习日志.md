##🔖 学习主题

LangChain 0.1 → 1.2 版本升级，OutputParser 与 structured_output 使用，Prompt 模板与 pipeline 理解

###1️⃣ Prompt 模板设计
####目的：构造 LLM 输入，让模型生成内容更可控
####示例：
```python
from langchain_core.prompts import ChatPromptTemplate

prompt = ChatPromptTemplate.from_template("{query}")
prompt.invoke({"query": "你好"})
```
####输出 PromptValue 或 messages
####作用：
#####变量注入：{query}、{language} 等
#####控制模型语言风格和输出格式
#####支持 pipeline 多变量输入
####版本差异：
*0.1：prompt.invoke(...) 返回 PromptValue，需要手动嵌套调用 LLM
*1.x：可用 chain = prompt | llm 替代嵌套调用
###2️⃣ LLM 调用与 AIMessage
####旧写法：
ai_message = llm.invoke(prompt.invoke({"query": "你好"}))
####作用：
*prompt.invoke 填充模板变量
*llm.invoke 调用模型，返回 AIMessage 对象
####AIMessage 结构：
|属性	|说明|
|:---------------:|:------------------:|
|content	|模型生成的文本（字符串）|
|additional_kwargs|	tool calling 信息|
|response_metadata|	token、日志等|
####嵌套调用问题：
*0.1 必须嵌套调用
*1.x 已用 pipeline (prompt | llm) 简化
###3️⃣ OutputParser 与 RunnableLambda
####旧方法：
*StrOutputParser → 从 AIMessage 提取字符串
*JsonOutputParser → 从字符串 JSON 提取 dict
####新方法：
*RunnableLambda(lambda x: x.content) ≈ StrOutputParser
*更灵活，但安全性和健壮性略低
*1.x 推荐使用 with_structured_output 替代两者
###4️⃣ structured_output（结构化输出）
*定义：LLM 输出约束，直接生成 Pydantic 对象
*示例：
```python
from pydantic import BaseModel
from langchain_openai import ChatOpenAI

class Joke(BaseModel):
    joke: str
    category: str

llm = ChatOpenAI(model="gpt-oss-120b").with_structured_output(Joke)
result = llm.invoke("讲一个程序员笑话")
print(result.joke, result.category)  # 返回 Pydantic 对象
```
*优点：
**自动 JSON → 对象解析
**类型安全，减少解析错误
**不再需要手动 OutputParser
*注意：
**必须提供 schema（Pydantic 类或 JSON schema）
**如果缺省 schema，with_structured_output() 会报错或退化为普通文本输出
###5️⃣ pipeline 与字典输入设计
原理：
*LangChain 1.x 所有节点输入统一为字典
*支持多变量、可复用 pipeline
示例：
```python
chain = prompt | llm | parser
result = chain.invoke({"query": "你好"})
```
数据流：
{"query": "你好"} → prompt (填充模板) → PromptValue → llm → AIMessage → parser → str
设计原因：
*支持多变量输入
*pipeline 节点统一接口
*灵活组合多个节点（retriever、工具调用等）
###6️⃣ 0.1 → 1.2 版本差异总结
|功能|	0.1 写法|	1.2 写法|
|:-----------:|：--------------------：|：-------------------------------------------：|
|Prompt 调用	|prompt.invoke({"query": ...})	|`prompt|
|OutputParser	|StrOutputParser / JsonOutputParser|RunnableLambda 或 with_structured_output|
|嵌套调用|必须|可省略，直接 pipeline|
|返回类型|AIMessage → 手动解析|structured_output → Pydantic对象|
###7️⃣ 核心学习总结
*1.Prompt 模板 = 构造 LLM 输入
*2.AIMessage.content = 模型文本输出，默认字符串
*3.OutputParser 旧方法 → 解析字符串/JSON
*4.structured_output → 直接生成结构化 Pydantic 对象
*5.pipeline + dict 输入 → 统一接口，支持多变量与复合链路

###✅ 收获：
通过今天的学习，我理解了 LangChain 从 0.1 → 1.2 的架构升级逻辑，尤其是：

*字符串输出 → Pydantic 结构化输出
*手动 invoke → pipeline 自动化
*OutputParser 弱化 → structured_output 强化
