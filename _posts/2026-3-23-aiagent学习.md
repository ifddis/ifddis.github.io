## 🧠 核心知识点

### 🔹 1. PromptTemplate vs ChatPromptTemplate

* PromptTemplate：

  * 单字符串模板
  * 用 `format()` 填充变量
  * 无角色概念

* ChatPromptTemplate：

  * 多消息结构（message list）
  * 使用 `(role, content)` 形式
  * 支持 `"system" / "human" / "ai"` 角色

---

### 🔹 2. 模板变量（{query}）

* `{query}` 是占位符，用于动态替换用户输入
* 通过：

```python
prompt.invoke({"query": "xxx"})
```

进行绑定

👉 本质：**模板 + 数据分离**

---

### 🔹 3. 为什么 invoke 传字典

* 为了支持多变量：

```python
{"query": "...", "name": "..."}
```

* 与模板变量一一对应
* 提高复用性 & 可扩展性

👉 本质：**输入参数结构化**

---

### 🔹 4. human / system 的作用

在 ChatPromptTemplate 中：

```python
("system", "...")  # 设定模型行为
("human", "{query}")  # 用户输入
```

* `"system"`：定义模型身份/规则
* `"human"`：用户输入
* `"ai"`：模型输出（隐式）

👉 本质：**模拟真实对话上下文**

---

### 🔹 5. Runnable 机制（新版本核心🔥）

旧版本：

```python
prompt.format() → llm()
```

新版本：

```python
prompt.invoke() → llm.invoke()
```

甚至可以：

```python
chain = prompt | llm
chain.invoke({...})
```

👉 本质：

> **Runnable = 数据流管道（Pipeline）**

---

### 🔹 6. partial 的作用

```python
.partial(now=datetime.now())
```

* 预先绑定部分变量
* 后续调用不用再传

👉 本质：**默认参数注入**

---

### 🔹 7. PromptTemplate.from_template()

作用：

* 从字符串自动解析变量
* 生成 PromptTemplate 对象

---

## 🔄 LangChain执行流程（重点🔥）

```text
用户输入
   ↓
prompt.invoke({"query": ...})
   ↓
生成 PromptValue（消息列表）
   ↓
llm.invoke(...)
   ↓
返回 AIMessage
   ↓
ai_message.content（最终结果）
```

---

## 💻 代码实践

```python
from datetime import datetime
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

prompt = ChatPromptTemplate.from_messages([
    ("system","你是助手，现在时间是{now}"),
    ("human","{query}")
]).partial(now=datetime.now())

llm = ChatOpenAI()

ai_message = llm.invoke(
    prompt.invoke({"query":"现在几点，请讲一个程序员冷笑话"})
)

print(ai_message.content)
```

## 🐛 今日踩坑

### ❌ 问题1：模板报错

```text
ValueError: expected '}' before end of string
```

### ✅ 原因：

* `{query` 少了 `}`

---

### ❌ 问题2：输出结构变复杂

```text
content=... response_metadata=...
```

### ✅ 原因：

* 新版本返回 AIMessage 对象

### 💡 经验：

```python
print(ai_message.content)
```

---

### ❌ 问题3：模型调用报错

```text
429 / no available channels
```

### ✅ 原因：

* API 限流 / 模型不可用（多为代理问题）
